| **評価指標（KPI）**                | **実行管理・評価フェーズ**             | 実施した施策の進捗と成果をモニタリングし、戦略の実効性を確認・調整するための評価指標策定段階。 |

### 表2：GenAIでは代替できないステークホルダー・エンゲージメントの条件

| 条件 | GenAIが適さない理由 |
|------|---------------------|
| **成果に対するステークホルダーのコミットメントが求められる場合** | IT戦略に対する理解・共感・当事者意識を醸成するためには、策定プロセスにおけるステークホルダーの巻き込みが不可欠である。特に、現状との乖離が大きい戦略や多額の投資を伴う施策においては、初期段階からの合意形成が極めて重要。 |
| **インプット情報が未整備・非ドキュメント化である場合** | ステークホルダーとの対話から得られる示唆は、正式な資料に残されていない、もしくは形式知化されていない情報に基づくケースが多く、GenAIの分析対象外となる。そのため、暗黙知へのアクセスには人の介在が必要。 |
| **高度なドメイン知識や業務知見が求められる場合** | 一般的な情報は分析可能でも、企業固有の背景・業務プロセス・地域特性など、特定の関係者のみが保有する知識はGenAIでは補完困難。たとえば、ある機能が異なる地域でどう運用されているかなどの現場理解には、人の知識が不可欠。 |
| **イノベーティブな発想が求められる場合** | 組織横断的な知識・経験・視点の共有を通じて新たな発想を生み出すことは、イノベーションの源泉である。GenAIはそのプロセスの支援はできても、創造性や人間特有の直感に基づくアイディエーションを完全に代替することはできない。 |
